# rKYC 해커톤 시연 시나리오 v2.0

## 시연 개요

**시스템**: rKYC (Really Know Your Customer) - AI 기반 기업 리스크 시그널 탐지 시스템
**접근 방식**: 빈 화면 시작 → 실시간 시그널 생성 시연
**소요 시간**: 10-15분

## 시연 시나리오

### 오프닝 (1분)

> "안녕하세요. 오늘 소개드릴 시스템은 **rKYC** - Really Know Your Customer입니다.
>
> 은행 기업심사 담당자들은 매일 수십 개 기업의 리스크를 모니터링해야 합니다.
> 뉴스를 읽고, 공시를 확인하고, 재무제표를 분석하는 데 많은 시간이 소요됩니다.
>
> rKYC는 AI를 활용해 이 과정을 자동화하고, **리스크와 기회 시그널**을 실시간으로 탐지합니다."

---

### Scene 1: 빈 화면 시작 (1분)

**화면**: Signal Inbox (https://rkyc-wine.vercel.app/)

> "현재 시스템에는 아직 분석된 시그널이 없습니다.
> 빈 화면에서 시작해서, 실시간으로 시그널이 생성되는 과정을 보여드리겠습니다."

**액션**: 빈 시그널 목록 화면 표시

---

### Scene 2: 기업 선택 및 분석 실행 (2분)

**화면**: Demo Panel (우측 상단)

> "먼저 분석할 기업을 선택합니다. **엠케이전자**를 선택하겠습니다.
>
> 엠케이전자는 반도체 부품을 제조하는 코스닥 상장사로,
> 수출 비중이 높아 환율과 반도체 수출규제에 민감한 기업입니다."

**액션**:
1. Demo Panel에서 "엠케이전자" 선택
2. "분석 실행 (시연용)" 버튼 클릭

> "분석 버튼을 클릭하면, 백그라운드에서 AI 파이프라인이 실행됩니다.
>
> 파이프라인은 8단계로 구성되어 있습니다:
> 1. SNAPSHOT - 내부 데이터 수집
> 2. DOC_INGEST - 문서 처리
> 3. PROFILING - 기업 프로파일 생성 (DART + Perplexity)
> 4. EXTERNAL - 외부 뉴스/공시 검색
> 5. CONTEXT - 통합 컨텍스트 구성
> 6. SIGNAL - AI 시그널 추출
> 7. VALIDATION - 팩트체크 (Gemini Grounding)
> 8. INSIGHT - 최종 인사이트 생성"

**대기**: 30-60초 (진행 상태 표시)

---

### Scene 3: 시그널 생성 확인 (3분)

**화면**: Signal Inbox (시그널 목록 갱신)

> "분석이 완료되었습니다. 엠케이전자에 대해 **3개의 시그널**이 생성되었습니다."

**시그널 예시 설명**:

#### 시그널 1: DIRECT - 재무제표 업데이트
> "첫 번째는 **DIRECT 시그널**입니다.
> 2024년 3분기 실적이 발표되었고, 매출이 전년 대비 30% 증가했습니다.
> 반도체 부문 호조로 긍정적인 신호입니다."

#### 시그널 2: INDUSTRY - 반도체 산업 동향
> "두 번째는 **INDUSTRY 시그널**입니다.
> 글로벌 반도체 수요 증가로 관련 업종 전체가 호조를 보이고 있습니다.
> 엠케이전자에 **OPPORTUNITY** 영향으로 분석되었습니다."

#### 시그널 3: ENVIRONMENT - 정책/규제 변화
> "세 번째는 **ENVIRONMENT 시그널**입니다.
> 미국의 중국 반도체 수출규제가 강화되었는데,
> 엠케이전자는 중국 매출 비중이 낮아 **영향 제한적**으로 분석되었습니다."

---

### Scene 4: 시그널 상세 보기 (2분)

**화면**: Signal Detail Page

> "시그널을 클릭하면 상세 정보를 볼 수 있습니다."

**설명 포인트**:
1. **AI Reasoning**: "AI가 왜 이 시그널을 생성했는지 설명합니다."
2. **Evidence**: "모든 시그널에는 **출처(Evidence)**가 첨부됩니다. URL 또는 내부 데이터 경로가 표시됩니다."
3. **Fact Check**: "Gemini Grounding으로 사실 확인을 거쳤습니다. VERIFIED 상태입니다."

> "담당자는 시그널을 검토한 후 **검토 완료** 또는 **기각** 처리할 수 있습니다."

**액션**: "검토 완료" 버튼 클릭

---

### Scene 5: 기업 상세 페이지 (2분)

**화면**: Corporate Detail Page (엠케이전자 클릭)

> "기업 상세 페이지에서는 더 많은 정보를 확인할 수 있습니다."

**설명 포인트**:

#### DART 공시 정보 (100% Fact)
> "상단에는 **DART 공시 정보**가 표시됩니다.
> 대표이사, 설립일, 본사 주소, 결산월 등 100% 사실 기반 정보입니다."

#### 외부 정보 프로필 (AI 분석)
> "하단에는 **AI가 분석한 외부 정보 프로필**이 있습니다.
> 사업 개요, 수출 비중, 주요 고객사, 공급망 정보 등이 표시됩니다.
>
> 신뢰도 배지가 **HIGH**이면 공시 기반, **MED**이면 뉴스 기반,
> **LOW**이면 추정값입니다."

#### 정보 갱신 버튼
> "프로필 정보가 오래되었으면 **정보 갱신** 버튼으로 최신화할 수 있습니다."

---

### Scene 6: 두 번째 기업 분석 (2분)

**화면**: Demo Panel → 동부건설 선택

> "이번에는 **동부건설**을 분석해보겠습니다.
> 건설사는 현재 PF(프로젝트파이낸싱) 리스크와 금리 민감도가 높은 업종입니다."

**액션**: 동부건설 분석 실행

**예상 시그널**:
- DIRECT: 재무 현황 변화
- INDUSTRY: 건설업 PF 리스크
- ENVIRONMENT: 금리 정책 변화

> "동부건설에는 **RISK** 방향의 시그널이 더 많이 생성될 수 있습니다.
> 시스템은 **리스크와 기회를 균형있게** 탐지합니다."

---

### Scene 7: Anti-Hallucination 설명 (1분)

> "AI 시스템에서 가장 중요한 것은 **사실 기반 정보**입니다.
>
> rKYC는 **5단계 Anti-Hallucination 방어 체계**를 갖추고 있습니다:
>
> 1. **Soft Guardrails**: LLM 프롬프트에 단정적 표현 금지
> 2. **Number Validation**: 50% 이상 극단적 수치 자동 검증
> 3. **Evidence Validation**: 출처 URL/경로 실존 확인
> 4. **Entity Confusion Prevention**: 기업명 일치 검증
> 5. **Gemini Grounding Fact-Check**: Google 검색 기반 사실 확인
>
> 허위 정보가 생성되면 자동으로 **거부**되거나 **검토 필요** 플래그가 붙습니다."

## 예상 질문 및 답변

### Q1: 분석에 시간이 얼마나 걸리나요?
> "일반적으로 30초-1분 정도 소요됩니다.
> 8단계 파이프라인이 순차적으로 실행되며,
> 복잡한 기업일수록 조금 더 걸릴 수 있습니다."

### Q2: 어떤 LLM을 사용하나요?
> "Multi-Provider 전략을 사용합니다.
> - Primary: Claude Opus 4.5 (분석 품질 최우선)
> - Fallback: GPT-5.2 Pro, Gemini 3 Pro
> - 검색: Perplexity sonar-pro + Gemini Grounding"

### Q3: Hallucination은 어떻게 방지하나요?
> "5단계 방어 체계를 구축했습니다.
> 특히 Gemini Grounding으로 모든 시그널을 Google 검색 기반으로 팩트체크합니다.
> 허위로 판명되면 시그널이 자동 거부됩니다."

### Q4: DART 데이터는 어떻게 활용하나요?
> "DART OpenAPI를 통해 기업개황, 재무제표, 최대주주 정보를 가져옵니다.
> 이 데이터는 100% Fact로 표시되며,
> 주주 정보는 Perplexity 결과와 교차 검증합니다."

### Q5: 실시간 업데이트가 가능한가요?
> "현재 버전은 수동 분석 트리거 방식입니다.
> 다음 버전에서 Supabase Realtime을 통한
> 실시간 모니터링 기능이 추가될 예정입니다."

## 트러블슈팅

### 시그널이 생성되지 않는 경우
1. Worker 상태 확인 (Railway 대시보드)
2. Redis 연결 확인
3. API 키 만료 확인 (PERPLEXITY, GOOGLE, OPENAI)

### 분석이 오래 걸리는 경우
1. 네트워크 지연 가능성
2. LLM API 응답 지연 (30초 이상이면 refresh)

### 빈 화면이 계속되는 경우
1. F5로 페이지 새로고침
2. 브라우저 캐시 삭제
3. API 직접 호출로 데이터 확인
